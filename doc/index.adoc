The `rifts` crate is the canonical Rust server implementation of the **Rift Realtime Protocol (Rift/1)** —
a bidirectional messaging protocol for real‑time applications.  Rift/1 replaces raw WebSocket +
event‑string patterns with explicit semantics: eight message classes, offset‑based resume,
topic‑level QoS, schema‑first events, protocol‑level backpressure, and a pluggable persistence
layer.

== Crate Features

[cols="3m,7a",options="header"]
|===
| Feature | Description

| `websocket` *(default)*
| Standalone WebSocket transport (tokio‑tungstenite).  No framework dependency.

| `sled`
| Persistent embedded storage via `sled::Db`.  Enables `SledEngine`, `SledBroker`,
  `SledOffsetStore`, `SledLogStore`, `SledDedupeStore`, `SledSnapshotStore`.
  Restart the server and offsets/logs survive.

| `axum`
| Accept Rift/1 connections from `axum::extract::ws::WebSocket`.

| `actix-web`
| Accept Rift/1 connections from actix‑web WebSocket sessions.

| `warp`
| Accept Rift/1 connections from `warp::ws::WebSocket`.

| `ntex`
| Accept Rift/1 connections from ntex WebSocket handlers.

| `tokio-frameworks`
| Shortcut: enables `axum` + `warp`.

| `all-frameworks`
| Shortcut: enables `axum` + `actix-web` + `warp` + `ntex`.
|===

== Quick Install

[source,sh]
----
cargo add rifts
# With persistent storage:
cargo add rifts --features sled
----

== Quick Start

[source,rust]
----
use rifts::RiftServer;
use std::sync::Arc;
use tokio::sync::Notify;

#[tokio::main]
async fn main() -> rifts::Result<()> {
    let shutdown = Arc::new(Notify::new());
    RiftServer::builder()
        .websocket_transport()
        .build()?
        .run("127.0.0.1:9000".parse().unwrap(), shutdown)
        .await?;
    Ok(())
}
----
